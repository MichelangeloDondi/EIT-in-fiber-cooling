# External audit — D1 two-photon Raman repump: a third repump-line option, and a test of the line-independent floor

**How to use:** hand to an external auditor / fresh chat. Builds on the **settled** D1 program
(`D1_hybrid_consolidated_findings.md`, `EIT_brief_v11.md` §12, External Audits A & B) — do **not**
re-derive that physics. This evaluates one *new* option the existing audits did not consider: making the
**D1 repump coherent (two-photon Raman)** instead of incoherent. House style: concise, [V]/[I]/[O], cite
source+line, attack the [I] rows. **Compute, don't assert.** Gated on G1 (795 fiber) — no 795 guidance,
this is moot.

> **Calibration warning.** "Clever repump routing" has been wrong **twice** this session: the D2 EOM-Raman
> |2,+2⟩ clearer was floated as a swap-rescue and the master equation killed it (ideal-clearer floor ≈A;
> single-photon scatter wrecked the 93.5%-occupied dark state). The claims below are physically motivated
> [I] but **uncomputed**, and the *same* dark-state-scatter pathology applies here. Treat the upside as a
> hypothesis to falsify, not a result.

## Settled input — do not re-litigate
- **Pruning theorem [V]:** the cooling Λ must live on one line (same-line legs 6.835 GHz → one EOM
  bridges; cross-line 7.12 THz → comb-only). The existing framework then states "the other line takes only
  **incoherent** (repump) roles." **Note this is an *assumption*, not a theorem consequence** — a *same-line*
  D1 Raman repump (both legs on D1, 6.835 GHz apart) is coherence-feasible via one D1 EOM. So this option
  is **outside the assumed framework, not forbidden by it.**
- **Line-independent floor [V, but assumption-bound]:** GATE A coherent floor = **0.0072** [V]; realized
  ≈0.003 [I], repump-dominated. b_leak exactly 1/3 (m′=2), 2/3 (m′=0) on both lines (6j identity). **This
  "no D1 floor gain" verdict was derived assuming *incoherent* repumps** — each repump return is a
  spontaneous, recoil-paying event. That is exactly the assumption a coherent Raman repump breaks.
- **The recycler is repump-dominated [V, triple-confirmed]:** per leak event m′=0 N_cool≈3.0, **N_rep≈7.0**,
  ρ≈2.3 (D2 repump); full-D1 4.69/**13.16**/2.81. **The repump recoil is the dominant heating term** — so
  it is the term a recoil-free return would attack.
- **F′=1 branching inverts [V]:** D2 5/6:1/6 → **D1 1/6:5/6**, collapsing the σ⁻ funnel-back 5/12→1/12 →
  the D1-repump recycler is **1.65× worse**. This is the penalty a coherent Raman might *route around*.
- **No-broadening is a D1-line property, NOT a Raman property [V — corrects a prior overstatement]:** D1's
  5P₁/₂ (J=½) has **no tensor at all**, so a D1 repump — incoherent *or* Raman — carries none of the ~18 MHz
  F′=1 tensor broadening that the D2 repump has. **Raman adds no broadening benefit over D1-incoherent.**

## The proposal
Replace the **incoherent** D1 repump with a **coherent two-photon Raman** repump on D1 (same-line, one D1
master + 6.835 GHz EOM): coherently transfer leaked spectator population (m′=0: |1,0⟩, |2,−1⟩, |1,+1⟩) back
into the cooling cycle, instead of single-photon-pumping it to a D1 F′ and accepting the (inverted)
spontaneous branching.

## The two things to test (the second is the big one)
**Q-A — does it recover the F′=1-inversion traffic? [I]** A coherent Raman can deposit a spectator onto a
*chosen* re-entry leg, bypassing the inverted D1 spontaneous branching that costs the 1.65× recycler
traffic. Plausible, but quantify against the incoherent D1 repump.

**Q-B — does it LOWER THE FLOOR below the line-independent value? [I — challenges the settled verdict]**
The entropy is generated by the **leak** (|F′2,0⟩→spectators), which is unavoidable and line-independent.
But the **return** of a spectator (already in a definite sublevel) to the cooling cycle can be a *unitary,
recoil-free* Raman transfer rather than a recoil-paying spontaneous repump event. Since the recycler is
repump-dominated (N_rep≈7 ≫ N_cool≈3), making the returns recoil-free could drop the floor **below** the
0.0072/≈0.003 that the incoherent-repump analysis fixed. **If true, this is the first floor improvement in
the program and supersedes the "no D1 floor gain" finding.** If false, it is killed by Q-C.

## Mandatory carryover checks — the failure modes (model explicitly, do not assume away)
1. **Dark-state-scatter pathology [V — same mechanism that killed the D2 clearer].** The Raman beams'
   *single-photon* off-resonant scatter on the dark leg is broadband (∝Γ) and is untouched by two-photon
   selectivity. On D2 a 0.63 kHz dark-leg scatter rate raised ⟨n⟩ 10× via dark-state **depletion** (not
   recoil). The D1 Raman repump has this same term — model it (as `clock_parasitic_solve.py` channel (c):
   Δ-independent rate, Kramers-Heisenberg branched, absorption+emission recoil).
2. **The recoil-free claim is gated by that scatter [O — the crux].** "Recoil-free return" is
   thermodynamically *allowed* (entropy is in the leak), but only *realized* if the Raman scatter is
   negligible — which forces large intermediate detuning Δ_R → weak Raman → slow returns. The same
   scatter-vs-rate tension that the D2 clearer **lost** decides it here. Q-B and check-1 are coupled: the
   detuning that suppresses scatter is the detuning that weakens the recoil-free transfer.
3. **Multiple tones / complexity [I].** m′=0 has three spectators → several Raman pairs/tones → more
   hardware *and* more scatter sources (each tone's single-photon scatter adds to check-1). Cost: a
   coherent D1 repump needs a **D1 EOM**, so H1's cost rises from "bare DBR" toward H2's ~€6–12k.

## Dispositive questions
1. **Q-A:** D1-Raman repump recycler traffic vs incoherent D1 repump — does it recover the 1.65× inversion
   penalty? (Beat full-D1's N_rep≈13 back toward D2's ≈7?)
2. **Q-B (the one that matters):** does it lower the steady-state floor below the line-independent
   0.0072/≈0.003? Report ⟨n⟩ **and** Nf convergence (a lower ⟨n⟩ that diverges is not a pass).
3. **Q-C:** at the Raman strength needed for Q-A/Q-B, is the dark-leg scatter small enough that the EIT dark
   state survives — or does it deplete as on D2 (the 3–10× penalty)? Scan Δ_R for a window where returns
   are fast enough *and* scatter is small enough; report whether the window is empty (as it was on D2).
4. **Cost:** the D1-EOM hardware delta vs the incoherent D1 repump.

**Gate (reuse the D2 discipline):** with the Raman repump replaced by the incoherent D1 repump, the solver
must reproduce the settled D1-repump recycler (N_rep≈13 full-D1 / the H1 number) and the GATE A 0.0072.

## Definition of done — three outcomes, all valuable
- **Q-B yes, Q-C clean:** the Raman repump lowers the floor below line-independent → **first floor
  improvement in the program; supersedes "no D1 floor gain."** Re-verify (frame-conflict, Nf) — this
  overturns a settled, twice-audited verdict, so the bar is high.
- **Q-A yes, Q-B no:** it recovers the inversion traffic but not the floor → a *capture/rate* refinement of
  the "D1 repump" arm of the existing S5 tradeoff (D2-incoherent vs D1-incoherent vs **D1-Raman**), not a
  floor win. Fold into the S5 weighing.
- **Q-C window empty:** the dark-state scatter eats it, as on D2 → **D1-Raman repump closed**, the
  incoherent-repump tradeoff stands. State which constraint failed (scatter rate, or returns too slow).
